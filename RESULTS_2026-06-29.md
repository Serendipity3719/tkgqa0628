# RESULTS_2026-06-29 — TKGQA Navigation Agent Experiment Report

## 1. Experimental Overview

This report summarizes the evaluation of the TKGQA navigation agent under two configurations:

- **Fixed Baseline (blind, no reflection)**
- **Fixed + Reflection (targeted P3 reflection enabled)**

Evaluation setting:
- n = 100 questions
- routing_mode = blind
- reveal = none (no gold qtype provided to agent)
- model = deepseek-chat
- tool environment = Linux Codespaces (bash-based execution)

## 4. Key Findings

### Finding 1 — Routing is NOT the bottleneck
Routing accuracy remains stable (~95–96%). This confirms that:
- SKILLS.md routing design is effective
- qtype → skill mapping is sufficient

### Finding 2 — Primary bottleneck: entity resolution mismatch
Majority of errors are not reasoning failures but:
- string normalization mismatch
- multi-answer formatting mismatch
- gold/pred schema inconsistency

This dominates failure taxonomy (>50% of errors).

### Finding 3 — Execution layer instability exists
Non-trivial execution_error indicates:
- some bash/skill commands fail at runtime
- tool execution layer still partially unstable in Codespaces/Linux transition

### Finding 4 — Reflection is selectively effective
Reflection improves:
- first_last (+31%)
- equal (+13%)
- multi-answer reasoning cases

But does NOT fix:
- execution errors
- entity normalization issues

Thus reflection improves reasoning, not system grounding.
