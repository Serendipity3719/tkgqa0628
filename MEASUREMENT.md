# 测量协议 (Measurement Protocol)

> 2026-06-27 · 起因: S2b 暴露"n=100 单次整体对比被 LLM 运行间噪声(~±3-4pt)淹没,
> 无法验证 <5pt 的定向改动"。本协议规定**今后任何 skill 改动如何被可信地验证**。

---

## 0. 核心问题 (为什么需要协议)

- 本项目每个 skill 改动的真实效应 ≈ **2-4 题** (per 100)。
- 单次 n=100 运行的噪声(temp=0 但 DeepSeek 非确定)≈ **±3-4pt / 6-7 题来回翻**。
- → **效应被噪声淹没**。直接比两次跑的整体 acc = 自欺。

实证 (exp_subset 配对 McNemar, blind2 vs blind3 = S2b 改动):
| 子集 | A→B | 改对/改错 | 净 | **p 值** |
|---|---|---|---|---|
| ALL | 85%→84% | 6/7 | -1 | **1.000** |
| visit (S2b 靶子) | 83%→91% | 3/1 | +2 | **0.625** |
→ **连 S2b 在它自己的靶子子集上都不显著 (p=0.625)。** 之前"visit 74→91 单调上升"是
**独立三次跑的噪声错觉**; 同题配对检验才是真相。

单文件 Wilson 95%CI (n=100) 同样说明问题: ALL `[76%,90%] ±7pt`; 子集普遍 ±15-30pt。

---

## 1. 工具 (`exp_subset.py`)

- **单文件**: 按 qtype / answer_type / 问句关键词(visit/appeal/negotiate/...) / has_pivot /
  multi_answer 切**定向子集**, 报 robust 准确率 + **Wilson 95%CI**。
  ```bash
  python exp_subset.py traces_X.json
  ```
- **双文件(同题配对)**: **McNemar 精确检验** —— 只看翻转题(b=改对, c=改错), 去掉题间方差。
  ```bash
  python exp_subset.py traces_before.json traces_after.json --subset visit
  ```
  判据: **净>0 且 p<0.05** 才算定向改动真实有效。

---

## 2. 验证一个 skill 改动的标准流程 (SOP)

1. **机制优先 (primary)**: 用真实库数据证明改动在**具体失败题**上把错的变对
   (如 S2b 在 quid 3000023 上 `< Make_a_visit` = gold)。这是最强、零噪声的证据。
2. **同题配对跑**: 改动前/后用**同一批题**各跑一次 → `exp_subset A B --subset <靶子>` 看 McNemar。
   - **不要**比两次跑的整体 acc(噪声)。只信配对翻转 + p 值。
3. **n 要够**: 要在子集上检测真 +10pp、α=0.05、~80% 功效, 子集需 ~**80-150 题**。
   visit 占比 ~23% → 总 **n≥400-650**。小改动想看显著, 就得跑大 n 或多 seed 配对累加。
4. **多 seed 累加(可选)**: 同配置跑 k 次, 把 k 次的配对翻转合并做一次 McNemar, 等效放大 n。

> 一句话: **小改动不靠"整体涨了几点"证明(那是噪声), 靠"机制对 + 同题配对显著"证明。**

## 3. 当前基线 (reference points)

| 配置 | n | robust e2e | 备注 |
|---|---|---|---|
| oracle (全 gold) | 100 | 91% | 上界 |
| blind 原始 | 100 | 81% | facet-blind, 修复前 |
| blind (S2′+S2b) n=100 跑 | 100 | 84-85% | **正噪声excursion(偏高)** |
| **blind (S2′+S2b)** | **300** | **81% [76,85] ±4pt** | **真实基线; routing 94.6%** |

> n=300 基线 = 今后改动的对照锚点: 新改动同样 n=300 跑一次, 与本基线**同题配对** McNemar。

### ⚠️ n=300 基线揭示的关键事实 (协议立功)
- **S2′ 修了 routing(88%→94.6%, 真), 但 aggregate e2e 没动**: n=300 = **81%**, 与原始 blind(81%)持平!
  n=100 的 84-85% 是正噪声。→ **routing 本就不是 e2e 瓶颈**, 修它不涨总分。
- 真实瓶颈(n=300, CI 可用):
  | 子集 | acc | n | 性质 |
  |---|---|---|---|
  | ans:**time** | **99%** | 96 | ≈ 已解决 |
  | ans:**entity** | **73%** | 204 | **gap 在这** |
  | **before_after** | **57%** | 72 | 最弱 qtype; 路由已修, 残错=多答案不全+gold噪声 |
  | **multi_answer** | **67%** | 110 | 跨类完整性问题(返回子集而非全部) |
  | has_pivot | 64% | 128 | 枢轴日期 |
- → **下一个该攻的不是 routing/绑定, 而是"多答案完整性"(before_after/equal_multi 返回全部)**,
  且这些子集 n=72-110 足够大, 在 n=300 配对下能检出真改动。但注意其中混有 gold/题面噪声(非模型错),
  真实可提升空间 < 名义 gap。

## 4. 对已有结论的诚实修正

- **S2′ (routing) 是少数能被检出的改动**: routing 88%→97% (+9pp, ~9 题) 远超噪声, 且
  机制清晰(before_after 误路由 4→0)。**可信。**
- **S2b (visit) 机制正确但整体不显著**: 数据级证明对; 但 n=100 配对 p=0.625。
  **采纳依据 = 机制正确 + 不变坏**, 而非"涨了点"。
- **facet 价值 / parse 负结果 (S1/S2a)**: oracle 91 vs blind 81 (~10pt)、parse 77 这些
  **大效应**仍可信(远超噪声); 但其中 ±2-3pt 的细节(如 blind 81 vs qtype-blind 79)是噪声, 不可细抠。
